# AI 原生开发元规范 (Meta Spec) —— 规范驱动编程 (Spec Coding) 范式

**版本**：1.2 (Decoupled Edition)
**核心哲学**：意图即主权，实现即瞬态。

---

## 一、 核心逻辑 (Core Philosophy)
在 AI 原生开发环境下，软件开发的本质从“编写语法”升维至“定义语义”：
- **文档即代码**：规范（Spec）不再是过期的注释，而是最高等级的源代码。
- **逻辑即架构**：系统的稳定性不取决于代码的堆砌，而取决于逻辑约束的严密性。

## 二、 核心要素 (The Core Elements)

### 1. 规范 (Spec)：结构化的意图 (Structured Intent)
- **本质**：规范是人类意志的逻辑化表达，是系统的“单一事实来源（SSOT）”。
- **定义**：由人类编写或由 AI 编写并经人类确认的，用于规约 AI 编程方向的约束性文档。
- **交付物示例**：产品需求文档（PRD）、系统架构设计、接口契约、状态机定义、协作协议。
- **核心特性**：
    - **人类主权**：人类是规范的唯一最终决策者。AI 可以提供提案，但必须由人类定案。
    - **双向对齐**：对人类用于交流业务逻辑，对 AI 用于生成可执行代码。
    - **溯源一致性**：严禁在代码层进行手动修改。任何变更必须回溯至 Spec。
- **物理隔离 (Physical Decoupling)**：在项目根目录中创建 `.spec/` 目录，项目中所有的 Spec 必须存放在该目录下，实现规范与代码的物理级完全解耦。
- **工厂映射**：在 `agent-skill-factory` 中，`.spec/` 及其子目录下的文件是最高级别的 Spec 实体。

### 2. 代码 (Code)：瞬态的实现 (Transient Implementation)
- **本质**：代码是意图在特定技术栈下的“折叠”和“投影”，是可丢弃、可再生的中间产物。
- **定义**：基于规范生成的、供计算机执行的指令集或供 AI 进一步处理的提示词。
- **交付物示例**：Java/Python 源码、AgentSkill 逻辑、OS 指令、配置文件、说明文档。
- **核心特性**：
    - **AI 所有权**：AI 是代码编写与逻辑验证的第一负责人。
    - **环境受限**：代码的正确性必须由测试环境闭环验证，无法自动验证的部分必须请求人类协助。
    - **单向投影**：从 Spec 到 Code 的转换必须是自动、幂等且单向的。
- **工厂映射**：在 `agent-skill-factory` 中，`projects/*/assets/` 及其生成的 `SKILL.md` 均被视为瞬态投影。

## 三、 双轮驱动架构 (Dual-Wheel Drive)

### 1. 规范驱动 (Spec-Driven)：做正确的事
- **原则**：先规范，后代码。严禁在没有规范定义的情况下开始编码。
- **同步性**：代码变更必须追溯至规范更新，保持两者同频。

### 2. 验证驱动 (Validation-Driven)：把事做对 (Agentic TDD)
- **原则**：在实现逻辑之前，先定义验证逻辑（状态探针）。
- **流程**：遵循“探针红（状态异常） -> 探针绿（状态恢复） -> 逻辑固化”的循环，由 AI 自动完成环境检测与代码修复。

## 四、 分工与协作协议 (The Protocol)

| 角色 | 负责领域 | 核心职责 |
| :--- | :--- | :--- |
| **人类 (Human)** | **规范 (Spec)** | 定义边界、决策风险、确认最终逻辑 |
| **AI (Agent)** | **编程 (Coding)** | 编写代码、执行验证、提出规范优化建议 |

### 开发流水线：
1. **意图输入**：人类定义初始规范。
2. **规范定稿**：人类审查并冻结 Spec 语义。
3. **代码投影**：AI 基于 Spec 全自动生成实现代码（通过编译器注入资产）。
4. **闭环验证**：AI 执行环境探针测试，失败则自迭代；环境受限则提交**规范修改提案**请求人类协助。

## 五、 反模式与红线 (Anti-Patterns & Red Lines)
为了捍卫 Spec Coding 的纯洁性，所有参与工厂开发的开发者必须遵守以下绝对红线：
1. **篡改投影 (Tampering)**：严禁手动修改生成的成品 `SKILL.md`。所有变更必须发生在 `.spec/`，并通过工厂工具重新投影。
2. **模糊语义 (Ambiguity)**：规范中严禁出现模糊词汇。Spec 必须是对 Agent 具有确定性指导意义的高密度指令。
3. **脱离上下文补丁 (Context-Blind Patching)**：严禁 AI 在未完全阅读并理解 `.spec/` 全局逻辑的情况下提供局部代码补丁。

## 六、 反馈闭环与持续演进 (Möbius Loop)
- **运行时反馈**：收集日志与用户反馈，自动评估效能。
- **规范建议**：当 AI 发现规范存在逻辑冲突或执行效率低下时，应主动发起修改提案。
- **持续演进**：通过“反馈 -> 修正规范 -> 重新投影代码”实现系统的持续进化。
