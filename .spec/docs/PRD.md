# agent-skill-factory 需求文档 (PRD)

## 1. 项目背景 (Background)
在 AI Agent 开发中，Agent Skill (技能) 是扩展 Agent 能力的核心。然而，目前 Skill 开发面临碎片化、依赖丢失、历史冗余、进化不透明等问题。
本项目旨在通过“工厂化”思路，构建一个通用的 Agent 技能生命周期管理平台。

## 2. 核心目标 (Goals)
- **多项目管理**：支持在一个工厂下并发开发多个不同领域的 Agent 技能。
- **动态进化**：建立一套“验证-确认-标注-记录”的标准化进化闭环。
- **双仓库同步**：实现开发源码与生产成品的分离，分别同步至 GitHub 仓库。
- **自包含交付**：成品 Skill (.md) 必须包含重建其依赖资产（如脚本、配置）的完整指令。

## 3. 核心需求 (Requirements)
- **R1: 源码工程化**：Skill 定义 (.tpl) 与物理资产 (assets/) 彻底分离。
- **R2: 铁律约束**：AI 必须遵循“禁止瞎猜 + 必须建议”的最高沟通原则。
- **R3: 零噪声交付**：生产环境的 Skill 文件必须剔除所有开发日志和历史。
- **R4: 自动化构建**：通过编译器自动注入 Heredoc 指令，实现资产的自愈能力。

## 4. 进化机制 (Evolution)
- **验证环节**：在用户反馈有效后，AI 模型标注 (如 `antigravity-gemini-3-flash`) 必须显式记录。
- **同步环节**：一键完成本地部署、项目同步和 Skills 分发同步。
