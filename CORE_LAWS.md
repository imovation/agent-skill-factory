# Agent Skill Factory - 核心铁律 (CORE_LAWS)

本工厂生产的所有技能及管理逻辑必须严格遵循以下铁律：

## 1. 沟通铁律 (Rule of Clarity)
- **禁止瞎猜**：若 AI 不理解用户描述或认为有歧义，必须先进行明确。
- **必须建议**：在请求明确时，必须基于专业判断提供 1-2 个建议方案及其优劣分析。

## 2. 进化铁律 (Rule of Evolution)
- **验证优先**：任何方案合入 Skill 前，必须经过用户真实环境验证。
- **显式确认**：验证有效后，必须询问用户是否确认进化。
- **模型标注**：每次进化必须标注执行任务的 AI 模型全名。

## 3. 工程铁律 (Rule of Engineering)
- **自包含性 (Self-Contained)**：生产的 `SKILL.md` 必须包含重建其依赖脚本（assets）的完整指令（Heredoc 格式）。
- **零噪声 (Zero Noise)**：生成的成品 Skill 文件严禁包含开发日志、调试历史或冗余描述。
- **双仓库同步 (Dual-Sync)**：进化完成后，必须同时同步“源码工厂仓库”和“成品技能仓库”。
- **安全隔离**：严禁在脚本或 Skill 文件中硬编码 Token 或敏感信息。
