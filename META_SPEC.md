# AI 原生开发元规范 (Meta Spec) —— 规范驱动编程 (Spec Coding) 范式

**版本**：1.0 (Official)
**核心哲学**：意图即主权，实现即瞬态。

## 一、 核心逻辑 (Core Philosophy)
在 AI 原生开发环境下，软件开发的本质从“编写语法”升维至“定义语义”：
- **文档即代码**：规范（Spec）不再是过期的注释，而是最高等级的源代码。
- **逻辑即架构**：系统的稳定性不取决于代码的堆砌，而取决于逻辑约束的严密性。

## 二、 核心要素 (The Core Elements)

### 1. 规范 (Spec)：结构化的意图 (Structured Intent)
- **本质**：系统的“单一事实来源（SSOT）”。
- **人类主权**：人类是规范的唯一最终决策者。AI 负责提供提案，人类负责定案。
- **溯源一致性 (Traceability)**：严禁在代码层进行手动修改。任何变更必须回溯至 Spec。

### 2. 代码 (Code)：瞬态的实现 (Transient Implementation)
- **本质**：意图在特定技术栈下的“投影”，是可丢弃、可再生的中间产物。
- **AI 所有权**：AI 是代码编写与逻辑验证的第一负责人。
- **单向投影 (One-way Projection)**：从 Spec 到 Code 的转换必须是自动、幂等且单向的。

## 三、 双轮驱动架构 (Dual-Wheel Drive)

### 1. 规范驱动 (Spec-Driven)：做正确的事
- **原则**：先规范，后代码。严禁在没有规范定义的情况下开始编码。
- **同步性**：代码变更必须追溯至规范更新，保持两者同频。

### 2. 测试驱动 (Test-Driven)：把事做对
- **原则**：在实现逻辑之前，先定义验证逻辑。
- **流程**：遵循“红 -> 绿 -> 重构”循环，由 AI 自动完成测试与修复。

## 四、 分工与协作协议 (The Protocol)

| 角色 | 负责领域 | 核心职责 |
| :--- | :--- | :--- |
| **人类 (Human)** | **规范 (Spec)** | 定义边界、决策风险、确认最终逻辑 |
| **AI (Agent)** | **编程 (Coding)** | 编写代码、执行验证、提出规范优化建议 |

### 开发流水线：
1. **意图输入**：人类定义初始规范。
2. **规范定稿**：人类审查并冻结 Spec 语义。
3. **代码投影**：AI 基于 Spec 全自动生成实现代码（如通过工厂编译器）。
4. **闭环验证**：AI 执行测试，失败则自迭代；环境受限则提交**规范修改提案**请求人类协助。

## 五、 反馈闭环与持续演进 (Möbius Loop)
- **运行时反馈**：收集日志与用户反馈，自动评估效能。
- **规范建议**：当 AI 发现规范存在逻辑冲突或执行效率低下时，应主动发起修改提案。
- **无限演进**：通过“反馈 -> 修正规范 -> 重新投影代码”实现系统的持续进化。
